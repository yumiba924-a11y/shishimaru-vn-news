## 0. このプロジェクトは何か（30秒で把握）
LSEG端末がCQCから無くなった。その制約下で、ベトナム株（VN30）の「動く図解新聞」を無料データで毎日作るのがゴール。
米国の人気アカウント「ぶたまる🐽米国株投資」の図解新聞をベンチマークに、ベトナム版を、より深く・動きをつけて作る。
運用環境は Claude Code (PowerShell 7) ＋ Claude Design。半自動（弓場さんの判断を1回通す）が前提。

## 1. 確定事項（検証済み・ここは動かさない）

**データソース = VNDirect 無料JSON API（Step 1で実アクセス確認済み）**
- 30銘柄の終値・前日比・騰落率・出来高 … `api-finfo.vndirect.com.vn`（finfo）。1コール・当日付・日本から直アクセスOK
- VN-Index / VN30 指数 … `dchart-api.vndirect.com.vn`（dchart）。finfoに指数が無いのでこちらで補完
- TCBS / SSI iBoard は404（仕様変更）で不可。Vietstock/CafeF のスクレイピングは予備に降格
- → 本命=VNDirect（finfo＋dchart）／予備=CafeF等スクレイピング

**ニュースソース（英語・無料・日次更新、Step後半で確認済み）**
- VnExpress International（e.vnexpress.net）… VN-Indexの日次の動きを毎日報道。図解の一言素材の主力
- The Investor（theinvestor.vn）… 機関投資家目線の深い記事。「狭く深く」路線に合う
- Vietstock English（en.vietstock.vn）… 中央銀行の政策・業種別ニュース
- 補助: Trading Economics / Investing.com … FTSE格上げ等の構造ネタの背景確認用

**市場構造の実測値（弓場さんの手元データ VN622.xlsx で計算）**
- VN30の30銘柄 = HOSE全体（403銘柄）の時価総額の 69.4%
- うち上位10銘柄だけで 49.8%（ほぼ市場の半分）
- → 「30銘柄を押さえれば市場の約7割を語れる」が自前データで確定

**VN30構成銘柄（2025年10月 CPVN資料準拠／年2回入れ替えあり・要更新）**
ACB, BCM, BID, CTG, DGC, FPT, GAS, GVR, HDB, HPG, MBB, MSN, MWG, PLX, POW, SAB, SHB, SSB, SSI, STB, TCB, TPB, VCB, VHM, VIB, VIC, VJC, VNM, VPB, VRE
※ VN30は1月・7月の第4月曜に入替。設定ファイル化し手で更新可能にすること。
→ 実装では `config/vn30_universe.json` に分離済み（セクター分類つき）。

## 2. コンセプト（昨日の議論で確定した「中身の作り方」）

**核となる見せ方** — 「上位10銘柄で市場の半分、30銘柄で7割を、日々の値動きで"要因分解"のように見せる」
- 指数が動いた → 誰が引っ張ったか → その内訳、と順を追って分解
- ベトナム市場が少数銘柄に偏っている（構造的弱点）を、図解では逆に強みに変える
- 米国は銘柄が多すぎて要因分解しにくい。ベトナムだからこそ成立する見せ方

**日次と週次の役割分担（"反復問題"への対処）**
- 課題: ベトナムは銘柄もテーマも少なく、同じヘッドラインが繰り返しがち
- 対処:
  - 日次 = 数字とタイル（定点観測）に徹する。毎日変わる数字が主役。ニュースは「分かりやすいものに絞る」
  - 週次 = 「なぜそうなったか」の深い解説を1本にまとめる
  - 同じ銘柄でも切り口を変える（値動き／需給・外国人枠／ファンダ／FTSEフロー）と反復に見えない
- → ぶたまるが「日次速報」で勝負する土俵に乗らず、「日次は定点・週次は洞察」でずらす

**差別化の核 = アニメーション**
- Claude Design はアニメーションを実際に動くHTML/CSS/JSとして出力できる（CSSキーフレーム、Chart.js等）
- 「指数がカウントアップ → 誰が引っ張ったか → 内訳が積み上がる」を一歩ずつ動かして見せる
- 希少性の三重の掛け算: 題材（ベトナム）× 表現（動く図解）× 専門知識（外国人枠・FTSE・構成比）
- 注意: ①アニメはPDF化で動きが消える（動く版=ブラウザ/URL、静止版=PDFの二段持ち）②トークン消費が重い

## 3. ツール分担
```
データ取得・自動化     = Claude Code（VNDirect API、毎日回す部分）
図解のデザイン・テンプレ = Claude Design（米国版を読ませてベトナム版テンプレ＋アニメ）
両者の橋渡し           = Claude Design の /design-sync・ハンドオフバンドル
```
- 一度テンプレが固まれば、毎日中身（数字）を差し替えるだけで動く図解が量産できる形を目指す

## 4. 出力レイアウト（設計書§5準拠・再掲）
```
ヘッダー（日付）
指数バー: VN-Index 終値±% / VN30 終値±%
今日のひとこと: 赤の大見出し＋3行（例「指数は重いが、銀行高・不動産安」）  ← 図解の主役
VN30タイル（30銘柄・緑=上昇/赤=下落/騰落率）＋ セクター別強弱
マクロ帯: ドン/円・USD/VND・金・原油（為替は使用レート明記＝社内ルール）
```

## 5. 進捗と次のステップ
設計書§8のステップ管理に対する現在地:
- Step 1（ソース検証）= **完了**（VNDirect確定）
- Step 2（取得→整形のスクリプト化）= **完了**（`scripts/fetch_vn30.ps1` ＋ `config/vn30_universe.json`）
- Step 3（図解HTML 1枚を出す）= **完了**（`scripts/render_zukai.ps1` → `outputs/vn30_<asof>.html`。静止版）
- Step 4（今日の一言の自動抽出）= **完了**（`scripts/interpret.ps1`。相場レジーム判定で見出し+3行を自動生成）
- Step 5（運用化）= **完了**（`scripts/run_daily.ps1` 1コマンドで [1]→[5] 通し。`-Open` でブラウザ起動）

**最初の達成目標「30銘柄の表から図解が1枚出る」= 達成。** 「端末なしでも毎日回る」を実証済み。
全面高/全面安/小動き/二極化の各レジームでテンプレが崩れないことを合成データで検証済み。

**リリース版ハードニング（2026/6/30 追加）:**
- マクロ帯をライブ化（USD/VND・ドン/円=open.er-api.com、金=gold-api.com、原油=手動）。per-item 縮退。
- 指数トレンドのスパークライン（直近20営業日）を指数バーに追加。
- 静止版 PNG/PDF を run_daily で自動書き出し（Edgeヘッドレス。Edge無くてもHTMLは出る）。
- 毎営業日 18:00(JST) の自動実行を Windowsタスクに登録（install/uninstall_schedule.ps1）。
- ガード: ①当日未確定スタブ（全銘柄0）を自動スキップし直近確定日を採用 ②基準日が古いと警告
  ③1月/7月 第4月曜のVN30入替が近いと警告。
- **マクロ/トレンド取得失敗は新聞本体を止めず該当ブロックのみ縮退**（弓場さん指示・検証済）。
**v2＝寄与度ドリブン（2026/6/30 実装完了。構成は `VN30_zukai_v2_kousei.md`）:**
- ④寄与度（綱引きの収支）を interpret 層に実装。contrib=ウェイト×騰落率（概算・明記）。
  発行済株式数を `config\constituents[].shares` に固定値保持（VNDirect OUTSTANDING_SHARES由来）。
- ②大見出し15字＋内容60字を寄与度から自動生成（米国版図解新聞と同型）。
- ⑥急変カード（|騰落率|≧3%の日のみ・上位6本・理由欄は半自動で人が1-2行）。
- ②指数バーのチャート/スパークラインは廃止し、紙面を寄与度に回した。
- 実証(6/29): VIC -1.3pt/VHM -0.4pt vs 銀行+0.5pt、ネット-1.1pt→実際-0.21%（概算ズレは明記）。
- 全レジーム＋寄与度/カード/マクロ欠落の縮退まで検証済。

**v3＝日次をぶたまる型に作り替え（2026/6/30。方針: 日次は今日のひとこと＋ニュースカードが主役）:**
- 日次レイアウト（render_zukai.ps1）: ①マストヘッド「DAILY QUEST｜VN30分解新聞」(CQC緑＋金) ②指数バー
  ③ステージ=VN30ヒートマップ(時価総額で大小=mcap_tランク)＋「今日のひとこと(大見出し15字/内容60字)＋ししまる(assets\shishimaru.png)」を重ねる ④ニュースカード4-5枚(主役) ⑤状態ラベル付きマクロ帯。
- **寄与度・セクター分解は日次描画から外し、週次(ウィークリークエスト)の主役に回す。** ただし interpret は寄与度を
  引き続き計算し interpreted.json に保持（週次素材として蓄積）。
- **ニュースカードは半自動**: `outputs\vn30_<日付>.news.json`（人がCafeF等の要約を記入・転載不可・`**赤太字**`/`[[緑]]`で強調）。
  interpret が±3%急変から雛形を自動生成（既存は上書きしない）。render はこの news.json を読む。
- マクロ状態ラベルは news.json の macro_labels 優先、無ければ前日比から自動判定。
- 参考レイアウトは弓場さん作 `Downloads\layout_butamaru_style.html`。米国版「ぶたまる」がベンチ。

→ 運用手順は `README.md`、v2寄与度の定義は `VN30_zukai_v2_kousei.md`（寄与度は週次の主役へ移管）。
**次フェーズ（Code側は完了済・残りは別ツール）= Claude Design で動く版テンプレを作り、出たHTMLを render に組込んで `interpreted.json` から量産化。** 渡す材料: v2構成doc＋`interpreted.json`＋v1/v2のHTML。ほか週次の深掘り。

## 6. 既知の弱点（正直に・引き継ぎ用）
1. 無料ソースは構造変更で突然壊れる → 取得層を独立させ差し替え前提（`fetch_vn30.ps1` だけ差替）
2. 日本からの地場サイトアクセス制限 → だからVNDirect API（JSON）を本命にした
3. VN30銘柄入れ替え（年2回・1月/7月第4月曜）→ リストを設定ファイル化（`config/vn30_universe.json`）
4. 前日比の基準（祝日・休場の処理）→ 整形層で明示的に持つ（`ref`＝前日終値、`prev_date` を記録済）
5. データの正確性は無料ソース由来 → 対外利用するなら出所明記と免責が要る
6. アニメのトークン消費 → 作り込みすぎ注意（Maxでも枠を食う）

## 7. スコープ外（今は触らない）

## 付録: 実装メモ（Step 2 実装時に確定した運用詳細）
- 取得層は **素のPowerShell 7（`Invoke-RestMethod`）／依存ゼロ**。Step 1 がこれで通ったため。Python(uv)は使っていない。
- 実行: `pwsh -NoProfile -File scripts\fetch_vn30.ps1`
- finfo は `change`/`pctChange` をソース側が算出済 → これを採用（祝日・休場の基準ズレを自前計算せず吸収）。基準の透明性のため `ref`(=close-change) と `prev_date` も保存。
- 出力JSONスキーマ: `as_of` / `indices{VNINDEX,VN30}` / `breadth{up,down,flat}` / `by_sector[]`（平均騰落率降順）/ `stocks[]`（騰落率降順・sector付）。Claude Design はこれを食えばよい。
- API仕様変更で finfo が空なら明示エラー終了 → 予備（CafeF等スクレイピング）は未実装。壊れたらここを差し替える。
