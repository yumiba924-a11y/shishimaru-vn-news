# VN30 図解新聞 自動化パイプライン — たたき台 v1
作成: 2026/6/29 / 前提: LSEG端末なし・無料公開ソース・Claude Code (PowerShell 7) 運用

---
## 0. このドキュメントの位置づけ
これは「動く保証」ではなく「作る順番と構造の合意」を取るためのたたき台。
実装段階で「サイト構造が想定と違う」は必ず起きる。その前提で、まず全体の絵を確定させる。

---
## 1. ゴール（今回のスコープ = B案）
毎営業日、VN30の30銘柄＋VN指数の「終値・前日比」を無料ソースから取得し、米国版「図解新聞」のベトナム版を1枚出力する。

**やること**
- VN指数（.VNI）の終値・前日比・出来高
- VN30構成30銘柄の終値・前日比
- 上記から「今日の一言（例: 銀行高・不動産安）」の材料を機械的に抽出
- 図解1枚（タイル＋見出し＋マクロ帯）を生成

**やらないこと（今回は捨てる）**
- 全402銘柄の取得（→ それは旧v4テンプレの役割。今回は不要）
- 週次トータルリターン・セクター加重（→ 後で足せる。初版は入れない）
- LSEG/Refinitivへの依存（→ 完全に切る）

## 3. アーキテクチャ（4段構成）
```
[1] 取得      data_fetch    : 無料ソースから30銘柄＋指数の生データを取る
      ↓
[2] 整形      normalize     : 旧v4テンプレの「抜粋」相当の綺麗な表に整える
      ↓
[3] 解釈      interpret     : 「今日の一言」「セクター別の強弱」を機械抽出
      ↓
[4] 出力      render        : 図解新聞HTMLを1枚生成
```
各段を独立させる理由：
- ソースが変わっても [1] だけ差し替えれば [2][3][4] は無傷
- [4] のデザインをいじっても [1][2][3] に触らなくていい
- 壊れたとき、どの段で壊れたか切り分けやすい

---
## 4. データソース戦略（「おまかせ」= 二段構え）
第一候補と予備を持ち、片方が落ちても止まらない設計にする。

| 優先 | ソース | 取得対象 | 想定リスク |
|---|---|---|---|
| 1 | 地場ポータル(Vietstock / CafeF) | 30銘柄の終値・前日比 | サイト構造変更・日本からのアクセス制限 |
| 2 | 海外系(investing.com 等) | 指数・主要銘柄の終値 | 無料枠の制限・取得整形の手間 |
| 補 | 手元xlsx(VN622形式) | 時価総額（シェア計算用・低頻度更新） | 手動更新 |

実装初日にやること：両ソースに実際にアクセスして、「30銘柄が一覧で取れるか」「日本から見えるか」を確認 → 安定した方を [1] の本命にする。

> v1以降の確定（引き継ぎメモ§1参照）: 実検証の結果、本命は **VNDirect 無料JSON API**（finfo＋dchart）に確定。Vietstock/CafeF は予備に降格。

## 5. 出力レイアウト（米国版図解新聞のベトナム版）
```
┌─────────────────────────────────────┐
│ ベトナム株 図解新聞      2026年X月X日      │  ← ヘッダー
├─────────────────────────────────────┤
│ VN-Index XXXX.XX (±X.XX%)  VN30 XXXX (±X%) │  ← 指数バー
├─────────────────────────────────────┤
│ [今日のひとこと]                          │
│ 「銀行に資金、不動産は調整」  ← 赤の大見出し   │
│  3行コメント                              │  ← 図解の主役
├──────────────────┬──────────────┤
│ VN30タイル(30銘柄)        │ セクター別     │  ← ヒートマップ
│ 緑=上昇 / 赤=下落 / 騰落率  │ 強弱サマリ     │
├──────────────────┴──────────────┤
│ マクロ帯: ドン/円 178  USD/VND 26,000  金 原油 │  ← 下帯
└─────────────────────────────────────┘
```
- 図解の主役は「赤い大見出し＋3行」。タイルは脇役（米国版と同じ思想）
- 為替は手元の基準値(178 VND=1円, 1USD=26,000VND)を初期値に。実運用では当日値に差し替え＋「使用レート明記」（社内ルール準拠）

## 6. 運用フロー（毎営業日）
```
1. Claude Code を起動（PowerShell 7）
2. fetch を実行 → 30銘柄＋指数を取得
3. 取得失敗があれば予備ソースに自動フォールバック
4. normalize → interpret → render
5. HTML 1枚が outputs に出る → 目視で1分チェック
6. OK なら完了（※「弓場さんの判断を1回通す半自動」設計）
```
ベトナム市場は日本時間17:00頃に引け。→ 夕方〜夜に回すのが自然。

## 7. 既知の弱点（正直に）
1. **無料ソースは構造変更で突然壊れる** … だから [1] を独立させ、差し替え前提にする
2. **日本からの地場サイトアクセス** … 制限がかかる場合あり。初日に要検証
3. **VN30銘柄入れ替え（年2回）** … リストを設定ファイル化し、手で更新できるように
4. **「前日比」の基準** … 祝日・休場日の処理を [2] で明示的に持つ
5. **データの正確性** … 無料ソース由来。社内/対外で使うなら出所明記と免責が要る

---
## 8. 作る順番（提案）
```
Step 1  ソース検証     : 地場/海外、両方に実アクセス。30銘柄取れる方を確定
Step 2  [1]+[2]       : 取得→整形まで。まず「綺麗な30銘柄の表」を作る
Step 3  [4]           : その表から図解1枚を出す（解釈[3]は最初は手動でOK）
Step 4  [3]           : 「今日の一言」抽出を自動化（最後でいい）
Step 5  運用化         : Claude Codeから1コマンドで回るようにまとめる
```
最初の達成目標は **Step 3 まで**＝「30銘柄の表から、図解が1枚出る」。
